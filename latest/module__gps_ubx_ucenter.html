Ublox GPS autoconfiguration

Module XML file: gps_ubx_ucenter.xml

Automatically configure any Ublox GPS for paparazzi.

  • configures all the messages, and the rates
  • automatic baudrate detection

Warning: you still need to tell the driver

  1. Which paparazzi uart you use
  2. Inside the ublox gps there are also many ports. The tiny/ppzgps use ublox_internal_port1 (UART1) but if for instance you use a LS-SAM or I2C device you need to set this by defining GPS_PORT_ID.

Module configuration options

Define Options

  • name: GPS_PORT_ID value: GPS_PORT_UART1
    Description: Port inside the Ublox GPS (GPS_PORT_DDC, GPS_PORT_UART1, GPS_PORT_UART2, GPS_PORT_USB, GPS_PORT_SPI)
  • name: GPS_UBX_UCENTER_RATE value: 250
    Description: Data output rate in ms
  • name: USE_GPS_UBX_RXM_RAW value: None
    Description: Activate raw measurments (only available on U-Blox T versions)

Init Functions

These initialization functions are called once on startup.

Periodic Functions

These functions are called periodically at the specified frequency from the module periodic loop.

  • gps_ubx_ucenter_periodic()
    • Frequency in Hz: 4.
    • Autorun: TRUE
      Periodic function automatically starts after init.
    • Start function: gps_ubx_ucenter_init()
      Executed before the periodic function starts.

<doc>

Warning: you still need to tell the driver

</makefile>
